This guide walks you through setting up CodesEverywhere on your local machine, from cloning the repo to sharing your first snippet.

## Prerequisites

You need the following tools installed before you start:

| Tool | Version | Purpose |
|------|---------|---------|
| Java | 17 | Runs the Spring Boot backend |
| Node.js | 18+ | Runs the React frontend |
| MySQL | 8.0+ | Stores snippets and user data |
| Maven | 3.8+ | Builds the Spring Boot project |

<Tip>
  Check your installed versions with `java -version`, `node -v`, and `mysql --version` before continuing.
</Tip>

## Setup

<Steps>

<Step title="Clone the repository">

Clone the CodesEverywhere repo and navigate into the project root:

```bash
git clone https://github.com/ANGELKORADIYA/codes-every-where.git
cd codes-every-where
```

The repository contains two main directories:

```
codes-every-where/
├── backend/       # Spring Boot application
└── frontend/      # React + TypeScript application
```

</Step>

<Step title="Set up the MySQL database">

Start your MySQL server, then open a MySQL client and create the database:

```sql
CREATE DATABASE 29db;
```

<Note>
  The database name `29db` is required — the backend is configured to connect to this exact database name. You do not need to create any tables manually; Spring Boot's `ddl-auto=update` will generate them on first run.
</Note>

Confirm your MySQL user has full access to the database:

```sql
GRANT ALL PRIVILEGES ON 29db.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```

</Step>

<Step title="Configure the backend">

The backend configuration lives at `backend/src/main/resources/application.properties`. The default values are ready to use if your MySQL is running on `localhost:3306` with the `root` user:

```properties backend/src/main/resources/application.properties
spring.application.name=codeseverywhere
spring.sql.init.platform=mysql
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.url=jdbc:mysql://localhost:3306/29db
spring.datasource.username=root
spring.datasource.password=root
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
```

<Warning>
  The default configuration uses `root` with password `root`. For any environment beyond local development, use a dedicated database user with a strong password and update these values accordingly.
</Warning>

If your MySQL setup differs from the defaults, update `spring.datasource.url`, `spring.datasource.username`, and `spring.datasource.password` before proceeding.

</Step>

<Step title="Run the Spring Boot backend">

Navigate into the backend directory and start the application with Maven:

<CodeGroup>

```bash Maven wrapper
cd backend
./mvnw spring-boot:run
```

```bash Maven (installed globally)
cd backend
mvn spring-boot:run
```

</CodeGroup>

The backend starts on port **8080**. You should see output similar to:

```
Started CodeseverywhereApplication in 3.4 seconds
```

Verify the API is running:

```bash
curl http://localhost:8080/api/
```

A JSON array of public snippets is returned (empty array `[]` on a fresh database).

<Tip>
  Keep this terminal window open. The backend must be running before you start the frontend.
</Tip>

</Step>

<Step title="Install frontend dependencies">

Open a new terminal, navigate to the frontend directory, and install dependencies:

<CodeGroup>

```bash npm
cd frontend
npm install
```

```bash yarn
cd frontend
yarn install
```

</CodeGroup>

</Step>

<Step title="Start the Vite dev server">

Start the React development server:

<CodeGroup>

```bash npm
npm run dev
```

```bash yarn
yarn dev
```

</CodeGroup>

Vite will print a local URL — typically `http://localhost:5173`. The frontend makes API requests directly to `http://localhost:8080/api` (hardcoded in the component files), so the backend must be running before you open the app.

<Note>
  Both servers must be running simultaneously: the Spring Boot backend on port 8080 and the Vite dev server on port 5173.
</Note>

</Step>

<Step title="Open the app and share your first snippet">

Open your browser and go to `http://localhost:5173`.

You will land on the **Dashboard** — a scrollable feed of public snippets from the community (empty on a fresh local install).

To share your first snippet:

1. Click **Share Code** in the navigation bar to open the code submission form.
2. Fill in the fields:
   - **Username** — your chosen username (a new account is created automatically on first use)
   - **Password** — a password for your account
   - **Title** — a short name for your snippet
   - **Type** — the language or category (e.g., `JavaScript`, `Python`, `Bash`)
   - **Code** — paste your code
   - **Description** — a brief explanation of what the code does
   - **Visibility** — set to **Public** to appear in the community feed, or **Private** to keep it to yourself
3. Click **Share Code** to upload the snippet.

<Check>
  Your snippet is now stored in MySQL and visible on the dashboard (if set to public). You can view all your own snippets under **Your Posts**, after authenticating with the same username and password.
</Check>

</Step>

</Steps>

## What's running

Once both servers are up, here is a summary of your local environment:

| Service | URL |
|---------|-----|
| React frontend | `http://localhost:5173` |
| Spring Boot API | `http://localhost:8080/api` |
| MySQL database | `localhost:3306/29db` |
