## Prerequisites

- **Node.js 18 or later**
- **npm** (bundled with Node.js)

Confirm your versions:

```bash
node --version   # should be v18 or later
npm --version
```

## Setup steps

<Steps>
  <Step title="Navigate to the frontend directory">
    The frontend lives in the `frontend/` subdirectory of the repository.

    ```bash
    cd codes-every-where/frontend
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    This installs all packages declared in `package.json`:

    | Package | Version | Purpose |
    |---|---|---|
    | `react` | ^18.3.1 | UI library |
    | `react-dom` | ^18.3.1 | DOM renderer |
    | `react-router-dom` | ^7.1.3 | Client-side routing |
    | `axios` | ^1.7.9 | HTTP client for API calls |
    | `lucide-react` | ^0.344.0 | Icon set |
    | `react-toastify` | ^11.0.3 | Toast notifications |
    | `tailwindcss` | (dev) | Utility-first CSS framework |
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Vite starts a local dev server, typically at **http://localhost:5173**. The terminal output shows the exact URL:

    ```
      VITE v5.x.x  ready in 300 ms

      ➜  Local:   http://localhost:5173/
      ➜  Network: use --host to expose
    ```

    The dev server supports hot module replacement — changes to source files reflect in the browser immediately without a full reload.
  </Step>
</Steps>

## Backend connection

The frontend communicates with the Spring Boot backend over HTTP. The base URL is hardcoded in the component files:

```
http://localhost:8080/api
```

<Warning>
  Make sure the backend is running on port 8080 before you open the frontend. If the backend is on a different port, update the base URL in the component files that construct API requests. A common pattern to manage this cleanly is to extract the base URL into a shared constant or environment variable.
</Warning>

## Building for production

<Steps>
  <Step title="Create a production build">
    ```bash
    npm run build
    ```

    Vite compiles and bundles the application into the `dist/` directory. The output is minified and optimized for deployment.
  </Step>

  <Step title="Preview the production build locally">
    ```bash
    npm run preview
    ```

    This serves the `dist/` folder with a local static file server so you can verify the production build before deploying.
  </Step>

  <Step title="Deploy the dist/ directory">
    Upload the contents of `dist/` to any static hosting provider (Netlify, Vercel, AWS S3 + CloudFront, etc.).

    <Note>
      CodesEverywhere uses client-side routing via React Router. When deploying to a static host, configure it to redirect all requests to `index.html` so that deep links resolve correctly.
    </Note>
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 5173 is already in use">
    Vite will automatically try the next available port (5174, 5175, etc.) and print the actual URL in the terminal. No configuration change is needed.
  </Accordion>

  <Accordion title="API requests fail with CORS or network errors">
    Confirm:

    1. The Spring Boot backend is running at `http://localhost:8080`.
    2. You can reach `http://localhost:8080/api/` directly in your browser or with `curl`.
    3. The `@CrossOrigin` annotation is present on the backend controller.

    If the backend is on a different host or port, update the API base URL in the frontend source files.
  </Accordion>

  <Accordion title="node_modules missing or stale">
    Delete `node_modules` and reinstall:

    ```bash
    rm -rf node_modules
    npm install
    ```
  </Accordion>
</AccordionGroup>
